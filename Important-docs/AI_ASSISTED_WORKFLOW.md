# AI-Assisted Workflow

## AI Tool Used

- ChatGPT GO for automation scripts
- Gemini 3 for test cases and documentation

---

## How AI Helped

AI was used to improve productivity during the assignment.

It assisted with:

- Creating the initial test plan structure.
- Brainstorming comprehensive test scenarios.
- Generating the first draft of Playwright test scripts through logical block of code.
- Suggesting repository structure and documentation format.
- Reviewing documentation for clarity and readability.

---

## Manual Review and Corrections

All AI-generated content was manually reviewed before being included.

The following changes were made after manual verification:

- Updated test cases to match the application's actual behavior.
- Corrected Playwright locators that did not match the application's DOM.
- Removed duplicate or unnecessary test scenarios.
- Refined bug report based on automation testing.
- Improved assertions to validate the expected behavior defined in the assignment specification.

---

## AI Limitations

AI accelerated documentation and test creation, but it could not accurately identify application-specific defects without manual testing.

Manual exploratory testing was required to:

- Discover actual defects.
- Verify expected behavior.
- Validate calculations.
- Confirm reproducible bugs.
- Assess defect severity.

---

## Summary

AI was used as a productivity aid rather than as a replacement for testing. All submitted test cases, automation scripts, and bug reports were manually reviewed, validated, and updated to ensure they accurately reflected the application's behavior and the assignment requirements.
